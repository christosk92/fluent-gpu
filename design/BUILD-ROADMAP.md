# fluent-gpu — Build Roadmap (design → gap-free implementation)

Generated by a 12-agent gap-analysis workflow (design spec vs actual `src/` code), synthesized into one
dependency-ordered plan. **Verdict: the current code is ~5% of the spec.** Reaching a gap-free framework is the
37 steps below, ordered strictly by dependency then value (shared contracts first). Steps marked **needs-pixels**
have correctness that can only be confirmed on-screen / with a real GPU or assistive tech — not headlessly.

**Critical path:** 1 (harness+sln) → 2 (Foundation allocators/handles) → 3 (Foundation value types/flags/deps) →
5 (Scene 64B/96B columns) → 6 (interning/realization) → 7 (SceneStore rebuild + dirty worklist + full ISceneBackend)
→ 8 (threading/publish seam shapes) → 9 (decouple D3D12 leaf) → 10/11 (DrawList+batcher; sRGB+offline DXC) →
12 (RHI handle table + PSO cache + device-lost) → 13 (Layout substrate) → 16 (ported-Yoga verbatim) →
17/18 (text measure seam + DirectWrite shaping leaf) → 20 (source-gen toolchain) → 21 (full reconciler) →
23 (13-phase loop) → 24 (render-thread split) → 25 (incremental layout) → 26 (pixel-snap) → 33 (parity/golden gates).

## How this is being executed
Subsystem-by-subsystem, each **implemented → headless-verified (new slice/golden checks) → committed**. Headless-verifiable
steps are proven by deterministic structural/golden assertions + the zero-alloc tripwire. **needs-pixels** steps require
your on-screen confirmation (and a real GPU / assistive tech), since this environment renders without a visible display.
