# FluentGpu Subsystem Design — COM Interop: Generated, Confined, Gated

*Subsystem design-of-record. Owns the deep design behind [hardened-v1-plan.md §4.2](../hardened-v1-plan.md) and [dotnet10-csharp14-zero-alloc.md §4 + §F](../dotnet10-csharp14-zero-alloc.md). This is the authority for: the ABI harvester + the two binding generators, `ComPtr<T>`/`IUnknownOps`/`CcwBox<T>`, the fail-safe `AbiVerify`, the generation-keyed `ComTracker`, the `FGCOM####` analyzer family, the cold-COM `[GeneratedComInterface]`-only policy, the residual hot-path hand-vtable surface, the non-suppressible CI-RELEASE COM gates, and ComPtr render-thread confinement + Move-only handoff.*

*References (does not duplicate) the shared contracts: threading/seam → [hardened-v1-plan.md §2/§4.1](../hardened-v1-plan.md); RHI/PAL seam → [architecture-spec.md §4.7/§5.1](../architecture-spec.md); text CCW → [text.md](./text.md) + [architecture-spec.md §5.3](../architecture-spec.md); UIA CCW → [architecture-spec.md §5.8](../architecture-spec.md); GPU renderer's ComPtr-in-leaf rule → [gpu-renderer.md](./gpu-renderer.md); language/AOT baseline → [dotnet10-csharp14-zero-alloc.md §1](../dotnet10-csharp14-zero-alloc.md).*

## 0. The one-paragraph thesis

COM is the single most dangerous hand-rolled surface in a NativeAOT, hand-`calli` engine: a wrong vtable slot index is undebuggable memory corruption, a leaked/double `Release` is a heisenbug, and a cross-thread refcount race is unprovable without a managed TSan. We retire each of those classes **structurally** wherever we can and **gate** the rest honestly:

1. **No human ever types a slot index or a calling convention.** Every hot-path binding is generated from a machine-harvested, checked-in `*.comabi.json` that is itself runtime-self-checked against the live system DLL.
2. **Exactly one thread (the render thread) owns every `ComPtr`.** Refcount races are not audited, they are impossible — there is only one writer of every refcount. ComPtr crosses the publish seam by **Move**, never by shared reference.
3. **All cold/warm COM is `[GeneratedComInterface]`/`[GeneratedComClass]` only**, isolated in `FluentGpu.PlatformIntegration` with its own marshalling assembly split. The hand-vtable surface shrinks to a named, justified residual.
4. **The shipping safety floor is CI, not the running app.** Every analyzer and tracker is `[Conditional]`-erased from the AOT binary, so production safety equals the coverage of a non-suppressible CI-RELEASE conformance/leak/ABI gate. We say so out loud.

The honest grade (carried from the hardened-v1 ledger): slot/convention is **SAFE-by-construction**; refcount *race* is **SAFE-by-construction** (confinement); refcount *lifetime* (leak/double-free) and the DWrite no-retain assumption are **MANAGED** (post-hoc detection + conformance test, not proof).

## 1. Where this subsystem lives (assemblies, threads, phases)

### 1.1 Assemblies

| Concern | Assembly | Kind | Marshalling attribute |
|---|---|---|---|
| `ComPtr<T>`, `IUnknownOps`, `CcwBox<T>`, `IID` constants, generated hot-path vtable structs | `Win32.Interop` (LEAF) | EXTRACT + NEW (generated `.g.cs`) | `[assembly: DisableRuntimeMarshalling]` |
| Generated D3D12/DXGI/DComp/DWrite-consume hot bindings | `Win32.Interop` (LEAF) | generated by `ComInteropGenerator` | inherits `DisableRuntimeMarshalling` |
| Cold/warm COM: UIA providers, TSF, OLE drag/drop, DWrite *setup* (factory/format/font-collection), WIC codec | `FluentGpu.PlatformIntegration` (LEAF) | `[GeneratedComInterface]`/`[GeneratedComClass]` only | **NO** `DisableRuntimeMarshalling` (source-gen COM uses `in`/`out` modifiers the attribute bans) |
| WIC image decode leaf | `Media.Codecs.Wic` (LEAF, behind portable `IImageCodec`) | `[GeneratedComInterface]` (cold, worker-thread) | follows the `PlatformIntegration` split rules (its own marshalling assembly) |
| The harvester (offline tool + CI step) | `FluentGpu.AbiHarvest` (build-time only; not shipped, not referenced by runtime assemblies) | NEW | n/a |
| `ComInteropGenerator` (consume) + `ComCalleeGenerator` (CCW) + `FGCOM####` analyzers | `FluentGpu.Interop.SourceGen` (Roslyn analyzer DLL, build-time only, referenced **only** by the Windows leaves) | NEW | n/a |
| `ComTracker`, `AbiVerify`, the CI conformance harness | `FluentGpu.Validation` + `Win32.Interop` (`[Conditional]`/CI-RELEASE-gated) | NEW | n/a |

**Acyclicity (enforced, from architecture-spec §3):** `FluentGpu.Interop.SourceGen` is referenced **only** by the Windows leaves, so the portable toolchain stays Win32-COM-free. The COM runtime types live below `Hosting`; nothing above the PAL/RHI/Text seam ever sees a `ComPtr`, `IUnknown*`, `HRESULT`, or `Guid` IID (architecture-spec §9 "what must NOT leak").

### 1.2 Threads (from hardened-v1-plan §2)

- **RENDER thread** — SOLE owner of every hot-path `ComPtr` (D3D12 device/queue/cmd-list/swapchain, DComp visual). All `AddRef`/`Release`/`Dispose`/`calli` on these happen here and only here. This is the keystone that makes refcount races *impossible*, not *audited*.
- **WORKER pool** — owns the WIC codec CCW/RCW for its own decode job *and only for the duration of that job* (decode → CPU bytes → handoff by handle). A worker's WIC `ComPtr` never escapes its `AllocScope`; the produced bytes are POD across the MPSC result ring. No worker touches a render-thread ComPtr.
- **UI thread** — touches **zero COM** (hardened-v1 §2.1). It owns no `ComPtr`, calls no vtable. The only COM-adjacent objects it touches are *cold CCWs* it cannot avoid: the UIA provider tree and OLE drag/drop callbacks, which the OS marshals **onto the UI thread** by contract (`ProviderOptions.UseComThreading` + the HWND pump; `DoDragDrop` pumps its own modal loop on the UI thread — architecture-spec §5.8). Those CCWs are therefore **UI-thread-confined**, a different confinement domain from the render-thread ComPtrs, and they never share an object.
- **DWrite shaping CCW** — the in-loop `IDWriteTextAnalysisSource`/`Sink` CCW is confined to **one** shaping thread (render thread in v1; a dedicated glyph-raster worker only after the phase-6 off-thread move). The DWrite **factory is SHARED and serialized** (see §9).

### 1.3 The 13-phase landing (which thread, which COM)

| Phase | Thread | COM activity owned here |
|---|---|---|
| 1 pump | UI | none. Reads device-lost reason word (Volatile, no ComPtr). |
| 2 input | UI | UIA *write* mutations posted; OLE drop callbacks (UI-thread CCW) routed into the input path. No render ComPtr. |
| 4 render / 5 reconcile / 6 layout | UI | none (zero COM). |
| 6.5 layout-effects | UI | none. |
| (worker, any time) | WORKER | WIC decode CCW lifetime (per-job, AllocScope-confined). |
| 8 record | RENDER | reads POD DrawList; no COM call yet. |
| 9 batch | RENDER | none. |
| 10 submit | RENDER | hand-vtable `calli`: `ID3D12GraphicsCommandList` record, `ExecuteCommandLists`, `Signal(fence)`. |
| 11 present | RENDER | hand-vtable `calli`: swapchain `Present1`, `IDCompositionDesktopDevice.Commit`, `IVideoPresenter.Place`. |
| 13 arena-swap / retire | RENDER | `ComPtr.Dispose` of fence-retired resources via `RhiHandleTable.DrainRetired` (deferred-delete ring). |
| 12 passive-effects | UI | none. |

DWrite text *setup* (cold `[GeneratedComInterface]`) happens at mount/format-change time on the UI thread's edge, never phases 6–13.

## 2. The ABI harvest pipeline: `FluentGpu.AbiHarvest`

The contract change folded from hardened-v1 §4.2: **the input to the binding generators is a checked-in `*.comabi.json`, not human-typed `[VtblIndex]` attributes.** No engineer ever writes the integer `9` for `ID3D12GraphicsCommandList::DrawInstanced`.

### 2.1 The triple-oracle harvest

`FluentGpu.AbiHarvest` is an offline tool (run by an engineer when pinning a new interface version, re-run in CI to detect drift). It harvests from three sources and reconciles them:

```
   Windows SDK winmd        d3d12.h / dwrite.h / dcomp.h ...        live system DLL
   (metadata projection)    (ClangSharp parse of the C++ ABI)      (runtime, via the self-check, §2.4)
            │                          │                                     │
            └──────────────┬───────────┘                                     │
                           ▼                                                  │
                  reconcile slot order,                                       │
                  calling convention, [PreserveSig],                         │
                  param blittability, IID, base-chain                        │
                           │                                                  │
                           ▼                                                  │
                  *.comabi.json  ◄──────── runtime self-check disposition ───┘
                  (CHECKED IN, content-hashed)
```

**Honest caveat folded from the critique:** winmd and ClangSharp are **NOT independent oracles** — they descend from the same header-level definitions, so they share header-level errors (a wrong base-chain in the SDK headers fools both). The harvest's *two static oracles agreeing* is necessary but not sufficient. The **only header-independent oracle is the live object** (§2.4 + §7). That is why the runtime QI-roundtrip is promoted to a non-suppressible CI gate, not a nice-to-have.

### 2.2 The `*.comabi.json` shape

```jsonc
{
  "abiVersion": 3,
  "harvestedFrom": { "winmdSha": "...", "headerSha": "...", "tool": "ClangSharp 18.x" },
  "interfaces": [
    {
      "name": "ID3D12GraphicsCommandList",
      "iid": "5B160D0F-AC1B-4185-8BA8-B3AE42A5A455",
      "baseChain": ["IUnknown", "ID3D12Object", "ID3D12DeviceChild", "ID3D12CommandList"],
      "firstOwnSlot": 10,              // slots 0..9 inherited (3 IUnknown + 4 Object + ... )
      "methods": [
        { "name": "DrawInstanced", "slot": 21, "preserveSig": true,
          "ret": "void",
          "params": [ {"t":"uint"}, {"t":"uint"}, {"t":"uint"}, {"t":"uint"} ],
          "blittable": true, "callConv": "MemberFunction" },
        { "name": "Close", "slot": 10, "preserveSig": false, "ret": "HRESULT", "params": [] }
      ]
    }
  ]
}
```

Key properties:
- **Slots are absolute and derived from the reconciled base chain**, not relative — the cross-assembly-inheritance vtable-offset pitfall (dotnet10 §6) cannot bite a hand-vtable consume binding because the JSON pins the absolute offset.
- `blittable: true` is asserted per method; the generator refuses to emit a hot-path `calli` for a non-blittable signature (it routes such a method to the cold `[GeneratedComInterface]` path or fails the build). This dovetails with `[assembly: DisableRuntimeMarshalling]` on `Win32.Interop` — the compiler *also* proves blittability (CA1421), giving belt-and-suspenders.
- The file is content-hashed; the hash is a CI input so a silent edit is detected.

### 2.3 What gets harvested

Only the surfaces we actually call, version-pinned (architecture-spec §5.3 pins `IDWriteFactory2+`, §5.1 pins the DComp set):
- **Consume hot path:** `ID3D12Device` (+ graphics fork slots), `ID3D12GraphicsCommandList`, `ID3D12CommandQueue`, `ID3D12Fence`, `IDXGISwapChain3`, `IDXGIFactory2/6`, `IDCompositionDesktopDevice`/`Target`/`Visual`, `IDWriteFactory2+`/`Font*`/`TextAnalyzer`/`GlyphRunAnalysis`.
- **Callee (CCW) ABI:** `IUnknown`, `IDWriteTextAnalysisSource`/`Sink`, `IDWriteGeometrySink`, `IRawElementProviderSimple`/`Fragment`/`FragmentRoot` + pattern interfaces, `IDropSource`/`IDataObject`/`IDropTarget`. The CCW generator needs the ABI to lay out the static vtable in the correct slot order — same JSON, consumed by `ComCalleeGenerator`.

### 2.4 The runtime self-check (closes the extraction-vs-runtime-ABI gap)

A SHA over the JSON proves the *file* didn't change; it does **not** prove the file matches the *DLL that actually loaded at runtime* (a serviced d3d12.dll could re-order, theoretically). Folded from hardened-v1 §4.5: **the generator emits a one-time, startup self-check that calls a known, side-effect-free method through the generated slot and asserts observed behavior against the live system DLL.**

```csharp
// Generated into Win32.Interop, run ONCE at device init on the render thread, CI-RELEASE-enabled, ship-stripped.
internal static AbiSelfCheckResult VerifyAbi(IUnknownOps ops)
{
    // 1. Anchor: QI(IID_IUnknown) on a live object must return the same pointer (identity law).
    // 2. For each generated interface: QI(ownIID) and assert pointer-equality / known E_NOINTERFACE.
    // 3. Behavioral probe per family: e.g. ID3D12Device.GetNodeCount() returns a small sane uint;
    //    IDWriteFactory2.GetSystemFontFallback() returns S_OK + non-null; swapchain.GetDesc1 fields match
    //    the SwapchainDesc we created. Compare observed-vs-expected; on mismatch -> hard fail (never ship a
    //    binding whose live behavior contradicts the JSON).
    return result; // logged + asserted by the com.aot.roundtrip SPIKE against the PublishAot'd binary
}
```

The behavioral probe is the header-independent oracle the static SHA cannot be.

## 3. The two generators (no human-typed slot or convention)

Both live in `FluentGpu.Interop.SourceGen` and consume the same `*.comabi.json` (passed as an `AdditionalFile`). They are incremental Roslyn generators reusing ComputeSharp's `EquatableArray`/`HierarchyInfo`/`DiagnosticInfo` discipline (architecture-spec §3b.3).

### 3.1 `ComInteropGenerator` (consume side — the hot path)

Emits one `readonly unsafe struct` per harvested interface wrapping a `T** lpVtbl`, with one method per JSON entry, each lowering to a `delegate* unmanaged[MemberFunction]<...>` `calli` at the **JSON-pinned slot**:

```csharp
// GENERATED — Win32.Interop, e.g. ID3D12GraphicsCommandList.g.cs
public readonly unsafe struct ID3D12GraphicsCommandList
{
    private readonly void** _vtbl;      // lpVtbl
    public static ref readonly Guid IID => ref Unsafe.As<byte, Guid>(ref MemoryMarshal.GetReference(s_iid));
    private static ReadOnlySpan<byte> s_iid => /* "..."u8 data-section, 0 alloc */;

    [MethodImpl(MethodImplOptions.AggressiveInlining)]
    public void DrawInstanced(uint vtxPerInst, uint instCount, uint baseVtx, uint baseInst)
        => ((delegate* unmanaged[MemberFunction]<void*, uint, uint, uint, uint, void>)_vtbl[21])  // slot from JSON
              (Unsafe.AsPointer(in this), vtxPerInst, instCount, baseVtx, baseInst);

    [MethodImpl(MethodImplOptions.AggressiveInlining)]
    public int Close()   // PreserveSig=false in JSON -> generator emits the HRESULT return + a thin throw helper at call sites that opt in
        => ((delegate* unmanaged[MemberFunction]<void*, int>)_vtbl[10])(Unsafe.AsPointer(in this));
}
```

- The integer `21` is *never* written by a human; it is the generator copying `methods[].slot` from the harvested JSON.
- `[MemberFunction]` calling convention is copied from the JSON, not chosen by a human.
- This is byte-identical to what `[GeneratedComInterface]` lowers to (dotnet10 §4) — so on the hot path it has zero downside and we keep control of the exact call site inside the devirtualized POD `SubmitDrawList` walk.

### 3.2 `ComCalleeGenerator` (CCW side)

Emits, for each interface we *implement* (DWrite analysis sink, UIA provider, OLE source), the static-vtable CCW recipe (architecture-spec §5.8 "PixelShaderEffect recipe"), again with slot order from the JSON:

```csharp
// GENERATED — the static vtable laid out in JSON slot order; methods are [UnmanagedCallersOnly] statics.
internal static unsafe class IDWriteTextAnalysisSource_Ccw
{
    private static readonly void** s_vtbl = BuildVtbl(); // QI, AddRef, Release, then own methods IN JSON ORDER

    [UnmanagedCallersOnly(CallConvs = new[] { typeof(CallConvMemberFunction) })]
    private static int QueryInterface(void* self, Guid* riid, void** ppv) { /* §6 fail-safe QI */ }
    // AddRef/Release route to CcwBox<T> (§4.3); own methods recover the managed impl via the box GCHandle.
}
```

The human writes only the *managed body* (`OnGetTextAtPosition`, `Navigate`, …). The vtable layout, slot order, `[UnmanagedCallersOnly]` plumbing, `this+offset` QI tear-off, and refcount wiring are all generated.

## 4. The three runtime primitives

### 4.1 `ComPtr<T>` — render-thread-confined, Move-only across the seam

Extracted verbatim from ComputeSharp (architecture-spec §3b.3) as the only lifetime primitive, then *hardened* with confinement and Move semantics:

```csharp
// Win32.Interop — unsafe struct, no GC alloc, AOT-trivial.
public unsafe struct ComPtr<T> : IDisposable where T : unmanaged
{
    private T* _ptr;

    public readonly T* Get() => _ptr;

    public void Attach(T* p) { /* take ownership, no AddRef */ }
    public readonly void CopyTo(ref ComPtr<T> dst) { IUnknownOps.AddRef(_ptr); dst._ptr = _ptr; } // shared ref -> AddRef

    /// MOVE — transfers ownership, leaves source null, NO refcount touch. The ONLY legal way across the seam.
    public ComPtr<T> Move() { var p = _ptr; _ptr = null; return new ComPtr<T> { _ptr = p }; }

    public void Dispose()
    {
        ThreadGuard.AssertRenderThread();           // [Conditional] — deterministic throw in asserts builds
        if (_ptr != null) { ComTracker.OnRelease(_ptr); IUnknownOps.Release(_ptr); _ptr = null; }
    }
}
```

- **Confinement:** every mutating op (`Dispose`, `CopyTo`, `Attach`) calls `ThreadGuard.AssertRenderThread()` — a `[Conditional("FGCOM_ASSERTS")]` deterministic throw. Refcount races are impossible because there is exactly one writer thread.
- **Move-only seam transfer:** the `SceneFrame` snapshot (hardened-v1 §2.3) is POD and carries **no** `ComPtr`. When a resource handle must change owners (it normally does not — the render thread owns them for life), it transfers via `Move()`, which touches no refcount and nulls the source, so two threads can never both hold a live copy. `CopyTo` (the AddRef path) is **forbidden across the seam** by FGCOM0004.
- **GC visibility:** `ComPtr<T>` roots live only inside the leaf `RhiHandleTable` (architecture-spec §4.7, P6) — the one permitted GC-edge for COM roots. Nothing on a per-frame path holds one.

### 4.2 `IUnknownOps` — the verified slot anchor

A tiny static surface that calls *only* the three ABI-frozen `IUnknown` slots (0=QI, 1=AddRef, 2=Release). These three slots are frozen by the COM standard across all of Windows history, so they are the one place where a hard-coded `0/1/2` is *defensible* and is the anchor `AbiVerify` (§6) builds on.

```csharp
public static unsafe class IUnknownOps
{
    public static uint AddRef(void* p)  => ((delegate* unmanaged[MemberFunction]<void*, uint>)(*(void***)p)[1])(p);
    public static uint Release(void* p) => ((delegate* unmanaged[MemberFunction]<void*, uint>)(*(void***)p)[2])(p);
    public static int  QueryInterface(void* p, Guid* iid, void** ppv)
        => ((delegate* unmanaged[MemberFunction]<void*, Guid*, void**, int>)(*(void***)p)[0])(p, iid, ppv);
}
```

### 4.3 `CcwBox<T>` — the callee-side managed root

Wraps a managed implementation behind a native struct the OS holds. Refcount is `Interlocked`; the `GCHandle` is freed **only** when the refcount hits 0 (the lifetime invariant the ledger labels MANAGED, backstopped by the leak gate §7).

```csharp
internal sealed unsafe class CcwBox<T> where T : class
{
    private GCHandle _self;            // Normal handle — keeps impl alive while native refs exist
    private int _refCount;             // Interlocked
    public void** Vtbl;               // points at the GENERATED static vtable
    public T Impl => (T)_self.Target!;

    public uint AddRef()  => (uint)Interlocked.Increment(ref _refCount);
    public uint Release()
    {
        int rc = Interlocked.Decrement(ref _refCount);
        if (rc == 0) { ComTracker.OnCcwFree(this); _self.Free(); /* NativeMemory.Free(native struct) */ }
        return (uint)Math.Max(rc, 0);
    }
}
```

For the DWrite shaping sink the box is a special, *synchronous, single-threaded* case: `AddRef`/`Release` no-op return 1 (DWrite does not retain across the analyze call — architecture-spec §5.3); the box instead **pins both** the source-state struct and the source UTF-16 `char[]` for the analyze-call duration. That no-retain assumption is the DWrite MANAGED residual, validated by the §9 concurrent conformance test, not proven.

## 5. `ComTracker` — generation-keyed, ABA-safe (CI-only)

`[Conditional]`-erased from ship; the localizer for net-refcount and leak failures. The critical fix folded from hardened-v1 §4.2: **key by `(pointer, generation)`, never raw `nint`.**

Why: snapshot slot recycling (and DComp/D3D12's own allocator) reuses native addresses. A raw-`nint` tracker sees address `0xABCD` freed and a *different* object reborn at `0xABCD`, and reports false-green (the AddRef/Release counts net to zero across two unrelated objects — classic ABA). Keying by a monotonically-bumped generation (mirroring the `Handle{index,gen}` discipline of foundations.md §4.1) makes the two epochs distinct ledger entries.

```csharp
internal static class ComTracker        // CI / asserts only
{
    private readonly struct Key(nint ptr, uint gen) { /* (pointer, generation) */ }
    // per Key: net refcount delta, alloc stack, owning thread id, alive flag.

    public static void OnCreate(void* p);          // gen = ++s_genFor[p]; new ledger entry
    public static void OnAddRef(void* p);
    public static void OnRelease(void* p);
    public static void OnCcwFree(object box);
    public static LeakReport SnapshotAtShutdown(); // any entry with net != 0 or alive==true -> fail
}
```

`ComTracker` also stamps the **owning thread id** on `OnCreate` and asserts on every subsequent op that the calling thread matches — this is what makes FGCOM0005 (cross-thread ComPtr) catchable at runtime in CI even where the static analyzer can't see through a forwarding shim.

## 6. `AbiVerify` — made fail-safe

The headline overclaim folded from hardened-v1 §4.2: **downgrade "impossible by construction" → "no human types the slot/convention; build-time cross-checked; runtime-verified in CI-RELEASE against live objects; shipping relies on generator correctness + the frozen-COM-ABI contract."** The verification itself must never become the thing that corrupts memory, so it is **fail-safe**:

1. **Never call AddRef/Release through an *unverified* slot.** Verification only ever dispatches through the three ABI-frozen `IUnknown` slots (`IUnknownOps`, §4.2) until a slot is proven.
2. **Anchor on the frozen IUnknown.** Identity law: `QueryInterface(IID_IUnknown)` on any live object must return a pointer that is itself QI-able back to the same identity.
3. **`QueryInterface(ownIID)` pointer-equality first.** For each generated interface, `QI(ownIID)` and assert the returned pointer relationship (equality for the canonical interface, known tear-off offset otherwise) **before** any own-slot call.
4. **Vectored-exception guard.** The probe runs inside an `AddVectoredExceptionHandler` scope so an access violation from a genuinely wrong slot is *caught and converted to a clean verification failure*, not an undebuggable crash. (Windows-only; the macOS equivalent is a Mach exception port, §10.)
5. **Behavioral probe** (§2.4) closes the loop against the live DLL.

`AbiVerify` runs at device init on the render thread, is `[Conditional]` in dev, **CI-RELEASE-enabled**, and stripped from ship. A failure aborts startup with a precise `(interface, slot, observed-vs-expected)` diagnostic rather than corrupting the heap.

### 6.1 Fail-safe QI in generated CCWs

The CCW `QueryInterface` (§3.2) is itself hardened, matching architecture-spec §5.3/§5.8: return `S_OK` **only** for `IUnknown` + the exact interfaces this CCW implements, else `E_NOINTERFACE`; the multi-interface tear-off uses `Unsafe.AsPointer(ref this) + offset` with the offset taken from the JSON layout, never a hand-counted byte.

## 7. The non-suppressible CI-RELEASE gates (the safety floor)

Stated bluntly (hardened-v1 §4.2/§8): **production safety equals CI coverage**, because every FGCOM analyzer and every tracker assert is `[Conditional]`-erased from the shipping AOT binary. If a developer suppresses the FGCOM analyzers under schedule pressure, **these gates are the floor that cannot be suppressed.** They run against the **`PublishAot`'d binary** (the `com.aot.roundtrip` SPIKE), not a debug JIT build, so they verify what actually ships.

| Gate | What it does | Disposition |
|---|---|---|
| **QI-roundtrip** | `AbiVerify` (§6) run against every harvested interface on a live device + the self-check behavioral probe (§2.4). The only header-independent oracle. | Non-suppressible; abort on fail. |
| **Net-refcount** | Run a representative frame workload; `ComTracker.SnapshotAtShutdown()` must report **0** net refcount delta per `(ptr,gen)`. ABA-safe by §5. | Non-suppressible; fail on any non-zero. |
| **Leak** | Allocate/free 10⁵ resources + CCWs (device-lost rebuild loop, virtualization recycle); assert every `GCHandle` freed and every native struct `NativeMemory.Free`'d. | Non-suppressible. |
| **ABI-golden** | The checked-in `*.comabi.json` content hash + the harvest re-run in CI must match; drift fails. | Non-suppressible. |
| **DWrite concurrency conformance** | The shaping-CCW + shared-factory concurrent test (§9). If it cannot be made green, off-thread raster is descoped (the named alternative). | Gating for the phase-6 off-thread move. |

These map to the validation spine's `com.aot.roundtrip` SPIKE + the per-PR COM net-refcount / leak / epoch GATES (hardened-v1 §4.5). The `__AbiSlots` self-check IL is **CI-only-stripped** — resolved as a CI-RELEASE gate and stripped from ship, *not* both trimmed-and-RELEASE-enableable (hardened-v1 §5).

## 8. The `FGCOM####` analyzer family (build-enforced; suppressible, hence backstopped by §7)

Lives in `FluentGpu.Interop.SourceGen`, referenced only by the Windows leaves. Each rule has a runtime CI backstop because analyzers are suppressible.

| Rule | Forbids | Why / backstop |
|---|---|---|
| **FGCOM0001** | A raw `IUnknown*` / `void*`-to-COM escaping a method return or field outside `Win32.Interop` | A raw pointer escape defeats `ComPtr` lifetime. Backstop: `ComTracker` alive-at-shutdown. |
| **FGCOM0002** | Unbalanced `AddRef`/`Release` (a `CopyTo`/AddRef without a matching `Dispose` on every path) | Refcount lifetime. Backstop: net-refcount gate §7. |
| **FGCOM0003** | Use-after-`Dispose` of a `ComPtr<T>` (dataflow: read after the `Dispose` call) | Use-after-free. Backstop: `_ptr==null` deterministic throw in asserts. |
| **FGCOM0004** | `ComPtr.CopyTo` (shared-ref AddRef) **across the publish seam**; only `Move()` is legal there | Two live copies on two threads → the race confinement is meant to prevent. |
| **FGCOM0005** | A `ComPtr` touched on a thread other than its owner (a method annotated render-thread-only called from UI/worker) | Cross-thread refcount. Backstop: `ComTracker` owning-thread-id assert (§5). |
| **FGCOM0006** | A hand-typed integer vtable slot or `[VtblIndex]` anywhere outside `IUnknownOps`'s frozen 0/1/2 | The "no human types a slot" contract. Backstop: ABI-golden gate. |
| **FGCOM0007** | `[GeneratedComInterface]` inheritance crossing an assembly boundary (the vtable-offset rebuild-coupling pitfall, dotnet10 §6) | Keeps each COM hierarchy single-assembly. |
| **FGCOM0008** | `ComWrappers`/`StrategyBasedComWrappers`/`GetOrCreateObjectForComInstance` on any per-frame path | The hot-path ruling (dotnet10 §4). |

The **transitive-forwarding gap** is honest: FGCOM0005 can't see a ComPtr laundered through a non-annotated helper. Closed (not by the compiler) by the `ComTracker` owning-thread assert firing in CI (hardened-v1 §4.5 `IsSpikeCaller`-style runtime backstop).

## 9. The residual hot-path hand-vtable surface (and why it's safe)

We do **not** eliminate hand-vtable `calli`; we eliminate *hand-typed slots*. The generated structs of §3.1 *are* hand-vtable `calli` — that is the point, per the dotnet10 §4 ruling (`[GeneratedComInterface]` compiles to the same `calli`; on the hot path it only adds a wrapper object + a cache lookup we don't want). The residual surface, all on the render thread, all generated, all `AbiVerify`-checked:

- **D3D12 consume:** `ID3D12GraphicsCommandList` record, `CommandQueue.ExecuteCommandLists`/`Signal`, `Fence`, swapchain `Present1` (phases 10–11).
- **DComp consume:** `IDCompositionDesktopDevice.Commit`, `IDCompositionVisual.SetContent`/`SetTransform` for the **multi-visual present tree** (UI swapchain visual z-above a video child visual; the transparent premul-0 hole-punch for `DrawVideoCmd.Dst`) and `IVideoPresenter.Place` (architecture-spec §5.1 amendment + WaveeMusic fold-in).
- **DWrite consume + the one in-loop CCW:** `IDWriteTextAnalysisSource`/`Sink` callee CCW during `Analyze*` (architecture-spec §5.3, text.md). The *factory* is SHARED and serialized; the shaping CCW is **thread-confined to the single shaping thread** (render thread in v1).

**Why it's safe:** (1) the slot/convention is generated and `AbiVerify`-checked, so no human-integer hazard; (2) it is all render-thread-confined, so no refcount race; (3) the CCW refcount is `Interlocked` and the GCHandle freed only at 0 (§4.3); (4) flat C exports (`D3D12CreateDevice`, `CreateDXGIFactory2`, `DCompositionCreateDevice`, `DWriteCreateFactory`, WIC factory create) are `[LibraryImport]`, not vtable dispatch — AOT-clean, blittable, no marshaller (dotnet10 §F).

**DWrite factory SHARED + serialized:** one `IDWriteFactory2+` is created once and shared; all access is serialized to the shaping thread. The concurrent conformance test (§7) either proves the shared-factory + per-thread-confined-CCW model holds, or **off-thread glyph raster is descoped** (the explicitly named alternative — hardened-v1 §4.2/§4.3). This is the gate for build-order phase 6.

## 10. Cross-platform (macOS) boundary

The whole COM surface is below the PAL/RHI/Text seam, so **nothing above `Hosting` recompiles** (architecture-spec §9). On macOS:

- `Win32.Interop` (COM, `ComPtr<T>`, the generators' COM output) is simply **not in the build**; the Cocoa/Metal/CoreText leaves replace it.
- The render-thread confinement *contract* survives unchanged: `id<MTLDevice/...>` and CoreText handles are owned by the render thread; the "single-writer of every native refcount" rule is identical, only the refcount mechanism is ARC/`retain`/`release` instead of `AddRef`/`Release`.
- The CCW pattern maps: DWrite analysis sink → CoreText has no callee callback of this shape (`CTFontGetGlyphsForCharacters` is a pull API), so the in-loop CCW *disappears* on macOS — a simplification. UIA providers → `NSAccessibility`-conforming objects reading the same `A11yInfo` columns (no static vtable; ObjC message dispatch).
- **The vectored-exception guard (§6.4) is Windows-only**; the macOS `AbiVerify` analog uses a Mach exception port + the same QI-equivalent identity probe against ObjC class identity. `AbiHarvest` is Windows-only entirely; the Metal/CoreText bindings are ClangSharp-against-the-ObjC-runtime, harvested into a parallel `*.macabi.json` at the CoreText milestone.
- A per-primitive Metal-milestone debt is tracked for anything that today depends on a Windows-only COM fallback (e.g. the D2D blur `IEffectRunner` crutch — hardened-v1 §4.3).

## 11. Failure / edge cases

- **Serviced system DLL re-orders a slot.** Caught by the runtime self-check (§2.4) at startup → clean startup abort with a diagnostic, never corruption. Frozen-IUnknown anchor + behavioral probe is the catch.
- **Wrong slot ships past CI** (the residual nightmare). Mitigated to "clean failure not corruption" in dev/CI by the vectored-exception-guarded `AbiVerify`; in **ship** it is the irreducible residual — production safety = the QI-roundtrip + behavioral coverage the team maintained. Stated honestly.
- **CCW refcount never reaches 0** (a native client over-AddRefs / leaks). The `GCHandle` and native struct leak; caught by the §7 leak gate in CI, post-hoc in ship (no detection). MANAGED.
- **DWrite retains the analysis source past the call** (violating the no-retain assumption). Would be a use-after-free of the pinned `char[]`. Caught only by the §9 concurrency conformance test; this is the named DWrite residual.
- **ABA on a recycled native address** → false-green leak report. Closed by `(pointer, generation)` keying (§5).
- **Device-lost mid-frame.** The render thread disposes every `ComPtr` (it owns them all) and rebuilds from CPU-retained state (architecture-spec §5.1). Because the UI thread holds no `ComPtr`, the rebuild needs no cross-thread refcount coordination. The device-lost wait callback runs on an OS wait thread and touches **no** `ComPtr` (Volatile.Write reason word only).
- **Cross-assembly `[GeneratedComInterface]` inheritance** silently mismatches vtable offsets after a partial rebuild → FGCOM0007 + the single-assembly `PlatformIntegration` policy.
- **Worker WIC CCW outlives its decode job.** Forbidden by `AllocScope` confinement + FGCOM0005; the produced bytes cross to the render thread as POD-by-handle, never the WIC `ComPtr`.

## 12. Zero-alloc & confinement summary

- **Hot path (phases 8–13):** generated `calli` through `T** lpVtbl` — zero managed allocation, zero RCW-cache lookup, zero boxing. IIDs are `"…"u8` data-section spans (no `const` runtime-constant, dotnet10 §F). The leak/refcount trackers are `[Conditional]`-erased → zero shipping cost.
- **Confinement domains:** render-thread (every D3D12/DXGI/DComp/DWrite-consume `ComPtr`, the shaping CCW); UI-thread (UIA + OLE CCWs the OS marshals there); per-worker (WIC decode CCW, AllocScope-scoped). No object is shared across domains; the only cross-seam transfer is POD `SceneFrame` + `ComPtr.Move()` (never `CopyTo`).
- **Cold path:** `[GeneratedComInterface]`/`[GeneratedComClass]` in `FluentGpu.PlatformIntegration` — a handful of long-lived wrapper objects at human timescale, never phases 6–13.

## 13. Honest residual

1. **Slot/convention: SAFE-by-construction** — no human types an integer; build-time cross-checked; runtime QI-roundtrip + behavioral probe against the live DLL. The one residual is a *shipped* wrong slot that escaped CI coverage, mitigated to clean-failure-not-corruption only in dev/CI.
2. **Refcount race: SAFE-by-construction** — render-thread confinement; impossible, not audited.
3. **Refcount lifetime (leak / double-free): MANAGED** — `[Conditional]` tracker + non-suppressible CI net-refcount/leak gates; post-hoc in ship.
4. **DWrite no-retain assumption: MANAGED** — validated by the concurrent conformance test, not proven; gates the off-thread-raster move.
5. **Analyzer suppressibility: MANAGED** — FGCOM rules can be suppressed; the floor is the non-suppressible CI-RELEASE conformance/leak/ABI gate. Production safety equals CI coverage, by design and stated.
6. **macOS `AbiVerify` parity is debt** — the vectored-exception guard is Windows-only; the Mach-port analog is a CoreText-milestone item.
