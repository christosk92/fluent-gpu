## 1. TL;DR

- **What is fragile/band-aided today.** The *value laws* are correct (channel enum, fold-to-`NodePaint`, spring velocity-handoff, dt-driven `Tick`, never-`LayoutDirty`), but everything *around* them is hand-stitched: tracks are heap `class Track` in a `List<Track>` with O(n) linear `Find`/`Get` scans (`AnimEngine.cs:69,585-601`) and a per-tick `Dictionary<NodeHandle,Accum>` scratch (`:166`); **six-plus independent phase-7 tickers** (`AnimEngine`, `InteractionAnimator`, `ConnectedAnimation`, `RepeatTicker`, `CaretBlinker`, `ScrollAnimator`, brush anims) each re-invent arm/disarm/idle-gate/`HasActive`; and **N uncoordinated writers race the same `NodePaint` columns**, avoiding collision only by *accident of disjoint node-sets and phase-7 call order* (`AppHost.cs:1121-1132`) â€” there is no enforced per-(node,channel) owner.
- **The ONE unifying idea.** Collapse the entire surface onto **a single POD primitive â€” an `AnimValue` slot keyed by `(NodeHandle, AnimChannel)`, carrying `{value, velocity, target, generator}` â€” that interpolates *from its own current value* and *auto-retargets on signal change*** (the CSS/SwiftUI/Compose "set-a-target, framework-owns-the-interpolation-from-current" model), evaluated by **one ordered frame scheduler** that owns the canonical `dt`, per-source *cadence*, and the *wake* decision. This is the `ScrollBind` slab idiom (`generic-hookable-scroll-engine-design.md` Â§3-4) generalized from "scroll-offset source" to "any signal source," with the animation generator (spring/inertia/keyframes) expressed as a pure `Eval(slot, t) -> {value, done}` POD â€” exactly Motion's generator contract sampled at *absolute* time.
- **Hard constraint 1 â€” zero managed alloc on phases 6-13.** Forces: tracks â†’ struct slab in `ChunkedArena` (no `class Track`, no `List` grow, no `Dictionary` scratch/`DrivenClockTable` closures); subscriber-lists â†’ the existing `TransformDirty|PaintDirty` mark; `params object[]` hook deps â†’ `DepKey` scalars; per-call `Keyframe[]` literals â†’ a shared keyframe arena.
- **Hard constraint 2 â€” dt-invariant determinism.** Forces the spring off sub-stepped semi-implicit Euler (`AnimEngine.cs:646-655`, whose sub-step count `n=clamp(ceil(dt/4ms),1,8)` makes the trajectory dt-path-dependent) and onto **Motion's closed-form analytical spring** sampled at absolute elapsed `t` (bit-stable across dt âˆˆ {8.33,16.67,33.3}), with the analytical `velocity(t)` derivative as the handoff primitive.
- **Hard constraints 3-5 â€” AOT/handle/SoA, render-thread/PUBLISH seam, transform/paint-only.** These are *already honored* by `AnimEngine`'s channelâ†’columnâ†’dirty-axis contract and validate the rework wholesale: namesâ†’handles once at reconcile; phase-7 writes only compositor columns (PUBLISH value-copies the bytes; render thread owns every `ComPtr`); the single sanctioned `LayoutDirty` escape (Reflow) stays an explicit opt-in slow path. **The animation engine is on the right side of every line WinUI's compositor model only reaches by convention** â€” the rework makes the partition structural.

## 2. The current model, precisely

### 2.1 The core: `AnimEngine` / `Track`

One `Track` is one integrator on one `(node, channel)` scalar. `AnimChannel : byte` is the fixed 17-entry vocabulary (`AnimEngine.cs:10`): `TranslateX/Y, ScaleX/Y, Rotation, Opacity, SizeW/SizeH, StrokeTrimStart/End, ClipL/T/R/B, LayoutW/LayoutH, BlurSigma`. `IntegrationMode : byte {Eased, Spring}` (`:15`); `CompositeOp : byte {Replace, Add, Accumulate}` (`:18`). `SpringParams` is a true 16B POD with the iOS/Compose-friendly `FromResponse(responseSec, dampingRatio)` ctor (`:21-34`). `Keyframe` is a POD record-struct (offset, value, per-segment `EasingSpec`, `:37-51`).

`Tick(dtMs)` (`:604-789`) is three passes â€” **advance** (per track, compute `Value`, set `Done`; spring = sub-stepped symplectic Euler `:646-655`; eased = wall-clock or driven `u`; `JustSeeded` skips first-frame integrate; delay-charge guard `:621-634`), **fold** (Replace tracks first, then Add/Accumulate so additive layers can't clobber the base, `:692-695`), **compose** (per touched node build `LocalTransform = Tâˆ˜Râˆ˜S`, write Opacity/Blur/Presented/Trim/Clip, mark `TransformDirty|PaintDirty` `:727`; the LayoutW/H reflow path feeds `LayoutInput` + marks parent `LayoutDirty` `:739` â€” the one sanctioned exception). The fold scratch is `Accum.FromPaint` (`:111-137`), which *preserves un-animated channels* by decomposing the live matrix (so a long scale track doesn't reset a settled opacity) â€” a real correctness win, but **lossy: rotation reads 0** (`:305`, "not cleanly recoverable from a scaled matrix") and scale uses raw `M11/M22` vs the polar `sqrt(M11Â²+M12Â²)` used in `FromPaint` itself (`:116-117`).

**Spec divergence (load-bearing for the rework):** the canon doc (`backdrop-effects-animation.md:403-421`) specs a single **64B `AnimTrack` POD in a `SlabAllocator<AnimTrack>`** with `ClockKind {Frame|Driven}`, `AnimFlags {Detached/PinsImage/Superseded/Completed/ReducedMotionExempt/AtRest}`, and a `NodeHandleâ†’TrackHead` O(1) map. **None of that is built.** The implementation is `class Track` + `List<Track>` + linear scans + `Dictionary` scratch + bool flags standing in for `AnimFlags`. The `DetachedAnimSlab` (Â§5.3) the spec owns for exit/detached animation **does not exist** â€” exit is done via live orphaned nodes (below).

## 5. The 'just works' principles (distilled, 8)

1. **Set a target; interpolate from current; auto-retarget on change.** The author declares a resting value (a signal); the engine reads the live value as the implicit "from" and, on any change, retargets the *existing* integrator â€” spring keeps velocity, eased re-seeds from current (optionally reverse-shortened). No manual "from," no channel bookkeeping. *(CSS transitions, SwiftUI `.animation(value:)`, Compose `animate*AsState`.)*
2. **Declarative enter/exit tied to mount + presence, not to imperative seeds.** Entrants seed from a starting snapshot; leavers defer their structural removal until their exit completes (a completion-gated detached snapshot). Presence is **ambient over a keyed child-set boundary** â€” no per-element opt-in. *(CSS `@starting-style`/`allow-discrete`, AnimatePresence.)*
3. **Spring-as-default with velocity handoff.** Interruptible motion is a spring carrying velocity across retargets (no kink on flip-back); fixed-duration "feel" uses a sampled `linear()` curve. *(SwiftUI/Compose default; CSS chose `linear()` over `spring()`.)*
4. **ONE value primitive + ONE frameloop, not N tickers.** A single `AnimValue` POD per (node,channel) and a single ordered scheduler owning the canonical `dt`, the per-source cadence, and the wake decision. *(Motion's `MotionValue` + `createRenderBatcher`.)*
5. **ONE composition point per (node, channel), ownership-partitioned.** Exactly one writer (or one compose pass) owns a nodeÃ—channel each frame; the partition is asserted, not assumed-by-disjointness. *(The `ScrollBind` R3 rule, generalized.)*
6. **Animation = data describing a transform, never imperative mutation.** A bound result is a value written into a compositor column (CSS effects-as-data / WAAPI composite-result / Framer MotionValue), evaluated by index over a slab â€” never app code mutating the scene per frame.
7. **Reduced-motion is a VALUE, never a conditional.** The engine reads it at tick/seed and snaps non-exempt motion to target while keeping opacity fades; authoring code never branches on it (so it can't change hook count). *(CSS media-query model; WCAG "reduce, don't remove.")*
8. **Signals drive targets; the per-frame path is index arithmetic over preallocated slabs.** Namesâ†’handles once at reconcile; a signal edge retargets a slab slot; lifetimes are reconciler-owned, not GC-reachable. *(The engine's own handle/SoA invariant, validated against every reference.)*

## 7. Key design tensions the DESIGN phase must resolve

**(a) Evolve `AnimTrack`/`AnimEngine` in place vs introduce a unifying `AnimValue` POD primitive.**
â†’ **RECOMMEND: introduce the `AnimValue` slab** (POD `{NodeHandle, AnimChannel, Position, Velocity, To, Generator, flags}` in a `ChunkedArena`, `(node,channel)â†’head` index, free-list lifetime) and re-express the existing tracks as rows on it. *Rationale:* the value laws are already correct, but the `class Track` + `List` + linear-scan + `Dictionary`-scratch model is the root of the alloc/throughput/stale-field footguns and is what the canon `AnimTrack`-in-`SlabAllocator` spec already mandated but no one built â€” the rework is the chance to land the spec, not re-patch the divergence.

**(b) The headline authoring API: declarative `animate`-prop/variants vs imperative Seed/Retarget vs signals-first `UseAnimatedValue`/`UseSpringValue` hook.**
â†’ **RECOMMEND: a layered combination with the signals-first hook as the spine.** `UseSpringValue(signal)`/`UseAnimatedValue(signal)` is the primitive (SwiftUI/Compose's converged ideal, maps 1:1 to retarget-keep-velocity); the declarative `animate=`/variant prop on `Element` is sugar that *bakes to that hook's seed at reconcile* (so the imperative recipe library routes through `Element.Animate`, deleting the `NodeHandle`/`OnRealized`/`UseLayoutEffect`/`Context.Anim`-null boilerplate that is the #1 app-ergonomics complaint â€” `LayoutTransition` already proves the declarative `Animate=` door works); imperative `Seed`/`Retarget` survives only as the low-level engine API the sugar lowers to. *Rationale:* one mental model ("declare a target, motion happens"), three altitudes, no fourth `UseAnimatedValue`-step-per-render vestige.

## Open questions for the design phase

1. **Compose pass vs owner-partition-only.** Is a single phase-7 fold-and-write-once pass worth the rewrite of the recorder's hover-scale composition (`SceneRecorder.cs:203-214`) and the sticky/scroll writers, or is an asserted owner column (cheaper, less invasive) sufficient? The compose pass deletes the `Accum.FromPaint` matrix round-trip but must preserve the hit-test-reads-layout invariant.
2. **Analytical spring under retarget = rebase or carry-state.** The closed form needs `(origin, vâ‚€, t=0)` recomputed on retarget (read current `value(t)`+`velocity(t)`, reset elapsed). Does this match the existing `ReframePosition` frame-shift handoff (`AnimEngine.cs:573`) exactly, and does the determinism gate tolerate the float-rounding difference between "rebase" and the current "mutate Target keep Pos+Vel"?
3. **Cadence vs vsync.** A per-source `Hz(n)` below panel refresh beats against the vsync-locked present (the SW wait stacks on vblank quantization). Can the present target a source's cadence (e.g. present-on-demand for driven sources) rather than a SW sleep, or is the sub-refresh beat an accepted limit documented like the other physical residuals?
4. **`relativeTarget` cost at list scale.** Motion's continuous parent-relative re-resolution is per-frame per-animating-node; bounded by `BoundsAnimated` count today, but a declarative `Layout` on a long (non-virtualized) list could make it O(visible rows) per frame. Is the depth-ordered pre-pass cheap enough, or does it need a "coherence group" cap?
5. **Presence boundary granularity.** Should ambient presence live on `ForEl`/`Show` (every keyed-child boundary gets it) or on an explicit `PresenceEl`? The former is more "just works"; the latter avoids surprising exit-deferral on lists that don't want it. Related: how does presence interact with the bound-row identity fix (Rank 4) so a re-skin never triggers exit?
6. **Generator keyframe storage.** A shared keyframe arena (`{offset,count}` span on the POD) needs a lifetime/recycling policy distinct from the track slab (static recipe keyframes are immortal; authored ad-hoc keyframes are per-mount). One arena with two regions, or two arenas?
7. **Reduced-motion exempt defaults.** Motion's selective model (transforms reduce, opacity exempt) vs a per-token explicit policy â€” what is the *default* `ReducedMotionPolicy` for an un-annotated `animate=` so cross-fades survive reduced-motion without every author opting in?
8. **Did the determinism gate ever cover springs?** The findings note the spring path may not be exercised by the dt-replay (or it would diverge under Euler). The design phase must confirm whether the analytical-spring switch *closes a latent gate hole* or merely *cleans up a passing-by-epsilon* case â€” this determines whether the switch is a correctness fix or a hygiene improvement.
